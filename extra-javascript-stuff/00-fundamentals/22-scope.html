JavaScript Concepts

Global and Local Scope

Last lesson we learned that when we write code in a browser it becomes a part of the global window object. Almost all the code we have written up until this point has been global. When something is global it means all code can access, change, and use it. This means we have to be very careful about creating anything that is global in scope so we don't accidentally do something we didn't intend.

When we create a variable inside of a function it can't be accessed from code outside of the function. Why? Because it's scope is local to that function, meaning it is invisible to any other code other than the other code inside of that function.

Understanding Scope

By understanding code's scope, we know when that code affects only one part of our code, or the entire codebase. If we create things that are global in scope, we are giving any code the supreme power to control our code. So we want to protect our code by being very careful when creating things that are global in scope. This is especially important if you plan on using JavaScript libraries like jQuery.

Local Scope

JavaScript uses function scope, meaning every time we create a new function the scope changes. Any code inside that function is local to that function. Code that is local in scope is not accessible to outside code.
